### 集合
①、Collection，主要由 List、Set、Queue 组成：
List 代表有序、可重复的集合，典型代表就是封装了动态数组的 ArrayList 和封装了链表的 LinkedList；
Set 代表无序、不可重复的集合，典型代表就是 HashSet 和 TreeSet；
Queue 代表队列，典型代表就是双端队列 ArrayDeque，以及优先级队列 PriorityQueue。

②、Map，代表键值对的集合，典型代表就是 HashMap

### List
```
ArrayList 动态数组
- 查询 
    - 时间复杂度O(1) 直接根据索引访问元素
- 插入 
    - 如果在列表末尾添加元素，时间复杂度为 O(1)。
    - 如果要在列表的中间或开头插入元素，则需要将插入位置之后的元素全部向后移动一位，时间复杂度为 O(n)。
- 删除
    - 如果要删除列表末尾的元素，时间复杂度为 O(1)。
    - 如果要删除列表中间或开头的元素，则需要将删除位置之后的元素全部向前移动一位，时间复杂度为 O(n)。
- 修改
    - 时间复杂度为 O(1)，直接根据索引修改元素
```

```
LinkedList 链表
- 查询
    - 是O(n)因为是链表要一个一个的查过去
- 插入
    - O(n),插入时间复杂度是O(1)但是需要先查一遍。。
- 删除
    - O(n),删除时间复杂度是O(1)但是需要先查一遍。。
- 修改
    - O(n),修改时间复杂度是O(1)但是需要先查一遍。。
```

### Map
```
HashMap 哈希表
数组 + 链表 + 红黑树

❌ 无序（插入顺序不保证）
✅ 查询快（O(1) 平均）
✅ 允许一个 null key
✅ 线程不安全
```

```
LinkedHashMap
HashMap + 双向链表

✅ 保持顺序（两种模式）：
- 插入顺序（默认）
- 访问顺序（LRU 用）
✅ 查询效率接近 HashMap
❌ 稍微多一点内存开销

(在访问顺序模式下，LinkedHashMap 需要维护“最近使用顺序”，
因此每次访问元素时都会将该节点移动到链表尾部，
以保证链表顺序始终反映访问顺序。

这种移动操作虽然是 O(1)，但会带来额外的性能开销。)
```

```
TreeMap
红黑树（自平衡二叉搜索树）

✅ 自动排序（按 key 排序）
❌ 查询比 HashMap 慢
❌ 不允许 null key
✅ 支持范围查询（很强）
```
