### 订单系统
1. 订单的流程是怎样的？
```
1.创建订单（校验库存 / 价格）
2.生成订单（状态：待支付）
3.锁库存（或预扣库存）
4.支付成功（回调）
5.扣减库存/确认库存
6.发货
7.完成/关闭（超时取消）
```
2. 如何防止重复下单？
```
1. 前端防抖（按钮禁用）
2. 后端幂等（订单号唯一约束）
3. 分布式锁（如 Redis 锁）
```

### 库存系统
1. 库存怎么扣？
```
1. 预扣库存（订单生成时锁定库存）
2. 实扣库存（支付成功后扣减库存）
3. 锁库存（订单生成时就扣库存（锁库存），支付成功后扣减库存，订单取消要回滚库存）
```
2. 如何防止超卖
```
乐观锁（version）
Redis 原子扣减（decr）
分布式锁（不推荐高并发）
队列削峰（MQ）
Redis 预扣 + DB 乐观锁 + MQ 异步处理
```
